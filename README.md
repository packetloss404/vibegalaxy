# vibegalaxy

A macOS native app ("Don't Anger the AI") that visualizes transcription/session data
as a living galaxy, tree, village, and nebula. The rendering runs in a mix of native
Metal (galaxy, particles, sphere preview) and an embedded Three.js scene (the tree/village
web view). The repo also contains an offline asset-generation pipeline for producing the
3D models used by the app.

## Repository layout

- `native-app/` — the Swift Package for the macOS app.
  - `Sources/DontAngerTheAI/` — app source (SwiftUI views, Metal renderers, data engine, web assets).
  - `scripts/` — model generation/decimation tooling.
- `generate-cosmic-mesh.mjs`, `generate-reference-image.mjs`, `generate-trellis-mesh.mjs` — root-level
  asset-generation scripts (Node/ESM).
- `render_sphere.py` — renders the sphere shader to a PNG (uses PyObjC + Metal).
- `scripts/compute-normals.py` — mesh normal helper.

## Building the native app

Requires macOS 14+ and a Swift 5.9 toolchain (Xcode).

```sh
cd native-app
swift build          # or: swift run
```

The package links the Metal, MetalKit, and QuartzCore frameworks and copies the
`Web/` directory (the Three.js scene) as a bundled resource.

## Asset-generation pipeline

The 3D models and reference imagery are generated by external APIs, not hand-authored.
Each script reads its API key from the environment (via a `.env` file, loaded with
`dotenv` for the Node scripts). Set the relevant keys before running.

Node scripts (run `npm install` in the repo root first):

| Script | Purpose | Env var |
| --- | --- | --- |
| `generate-reference-image.mjs` | Generate a reference image via Google Gemini | `GOOGLE_API_KEY` |
| `generate-cosmic-mesh.mjs` | Build a coarse mesh, then refine it with fal.ai UltraShape | `FAL_KEY` |
| `generate-trellis-mesh.mjs` | Generate a mesh from an image via fal.ai TRELLIS | `FAL_KEY` |

```sh
npm install
node generate-reference-image.mjs
node generate-cosmic-mesh.mjs cosmic-entity-reference.png
```

`generate-cosmic-mesh.mjs` first writes an intermediate coarse mesh to
`coarse-cosmic-entity.glb` at the repo root. This is scratch output and is git-ignored;
it is not committed.

Python model tooling (in `native-app/scripts/`):

| Script | Purpose | Env var |
| --- | --- | --- |
| `generate_models.py` | Generate app models via the Tripo 3D API | `TRIPO_API_KEY` |
| `decimate_models.py` / `decimate_glb.mjs` | Reduce GLB vertex counts (~10%) while preserving PBR materials | — |

Generated GLBs land in `native-app/Sources/DontAngerTheAI/Web/models/`.

## Notes

Never commit `.env` or API keys. See the `.gitignore` for ignored scratch/build artifacts.
